## 4. 当前存在的思维定势问题

### 4.1 过度依赖 "全静态网页 + JS动态渲染"
- 导航栏完全由JS生成：JS失败 -> 全站无导航
- 建议：服务端包含(SSI)或构建时模板化
- 或者: 至少有一个备用的静态nav

### 4.2 用文件层级替代构建工具
- 4层CSS @import + 11个JS文件 -> 7次HTTP往返
- 建议: 引入任意构建工具（Vite/Rollup/esbuild）
- 即使只用 cat 合并文件，也能减少请求

### 4.3 数据与视图未彻底分离
- calculator.html 中食物数据散落在HTML和JS中
- 271种食物数据在food-data.js，但旧的食物列表在HTML里
- 建议: 统一数据源，一个文件管理所有食物

### 4.4 "以防万一"的重复备份
- 根目录有3个动作库JSON文件、style.css.bak
- 建议: 清理冗余文件，用Git做版本管理

### 4.5 内联事件处理器（过时的交互方式）
- 大量 onclick/oninput 直接写在HTML属性中
- 混合了标记和逻辑，难以维护
- 建议: 统一事件委托 + data属性

### 4.6 无错误边界（脆弱架构）
- JS错误会直接导致页面白屏
- 无用户友好的错误提示
- 建议: 添加全局 error handler + fallback UI
